# RipCanvas Progress

## 已完成的功能点

- 支持从命令行打开 `.canvas` 文件：`rocv path\to\file.canvas`。
- 支持无参数启动，并显示空状态。
- 支持应用内通过文件菜单打开 `.canvas` 文件。
- 支持重新打开最近使用的 Canvas 文件。
- 支持手动刷新当前 Canvas 文件。
- 支持监听当前打开文件的外部变更，并在变更后自动重新加载。
- 支持缺失路径、非文件路径、非 `.canvas` 后缀和 JSON 解析错误的明确报错。
- 支持解析 Obsidian Canvas 的节点和边。
- 支持节点类型包括 `text`、`file`、`link`、`group`，未知类型使用安全回退显示。
- 支持节点标题、正文、文件路径、URL、颜色和几何信息的展示。
- 支持基础 Markdown 文本以 Slint `StyledText` 渲染。
- 支持 Obsidian Canvas 颜色值和十六进制颜色值映射。
- 支持根据节点背景色选择可读文本颜色。
- 支持渲染分组节点。
- 支持渲染曲线边，并根据节点连接侧生成边的起点、终点和控制点。
- 支持边端点颜色、渐变显示和流动波纹效果开关。
- 支持鼠标拖拽平移画布。
- 支持鼠标滚轮缩放画布。
- 支持缩放按钮、适配视图和 1:1 重置视图。
- 支持节点选择。
- 支持右侧 Inspector 查看选中节点的 id、类型、标题、标签、颜色、来源和几何信息。
- 支持复制选中节点信息到剪贴板。
- 支持折叠 Inspector。
- 支持小地图显示全局画布和当前视口位置。
- 支持开关节点标题显示。
- 支持开关边渐变、波纹效果和小地图。
- 支持快捷键打开文件、刷新、复制节点信息、适配视图和重置视图。
- 支持 Windows 图标资源和打包脚本。
- 支持从 CLI 导出 Canvas 为 PNG 图片：`rocv --export out.png path\to\file.canvas`。
- 支持在 UI 文件菜单和浮动工具栏中导出当前 Canvas 为 PNG 图片。
- 支持软件启动时自动最大化窗口。
- 支持以节点边的中点作为边连接的起点和终点。
- 支持点击节点后高亮相关联的节点和边。
- 支持顶部工具栏左侧锚定布局，品牌图标、标题和文件菜单不再居中漂移。
- 支持底部结构化状态栏，按状态、文件信息、画布统计和操作反馈分区展示。
- 支持 Inspector 未选中节点时展示当前画布摘要和快捷操作入口。
- 支持顶部栏和底部状态栏外层容器随窗口宽度铺满，避免整体布局随内容宽度居中。

## 未完成的功能点

### Canvas JSON 修改能力

- 暂未支持编辑 Canvas 节点内容。
- 暂未支持保存修改到 `.canvas` 文件。
- 暂未支持回写源文件。
- 暂未支持拖拽节点并保存节点位置。
- 暂未支持创建、删除或修改节点。
- 暂未支持创建、删除或修改边。

### UI 交互与展示变更

- 暂未支持在 Inspector 中修改 text 和 label 信息。
- 暂未支持自适应边布局算法来避免边被节点遮挡。
- 暂未支持完整 Obsidian Markdown 预览，包括嵌入、内部链接、任务列表和复杂块级语法。

### 其他功能与工程化

- 暂未支持多最近文件列表，目前只记录最近一个文件。
- 暂未支持大画布性能优化和虚拟化渲染。
- 暂未建立 GUI 自动化冒烟测试。
- 暂未完成跨平台安装包和自动更新流程。
